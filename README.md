## Russian – Word a Day (iOS)

This repo folder will hold the iOS app code for **Russian – Word a Day**.

### What’s implemented here
- **SwiftUI** screens: Main (search), Settings, Word Detail.
- **Offline dictionary** storage using **SQLite + FTS5** (built on first launch from a bundled JSON file).
- **Non-repeating daily word** logic (until the user resets used words).
- **Local notifications** with deep-link routing into the Word Detail screen.

### How to run (Xcode)
The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`, so opening Xcode is one step:

1. `open RussianWordADay.xcodeproj`
2. Pick an iPhone simulator (any iOS 17+ device).
3. Cmd-R to run. On first launch the app builds a local SQLite DB in Application Support and seeds it from the bundled `words.seed.json`.

If you ever need to regenerate the project (e.g. you added new files in `RussianWordADayApp/Sources/`):

```bash
brew install xcodegen   # one-time
xcodegen generate       # re-creates RussianWordADay.xcodeproj
```

Or build from the command line:

```bash
xcodebuild -project RussianWordADay.xcodeproj \
  -scheme RussianWordADay \
  -configuration Debug \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  build
```

### Data pipeline
- The app currently seeds the DB from `RussianWordADayApp/Resources/words.seed.json` (small sample).
- To build the ~5,000-word offline dataset, use the scripts in `scripts/` (see `scripts/README.md`).

### Notifications
- iOS Simulator supports notifications, but behavior varies by version.
- On device: ensure iOS **Developer Mode** is enabled and notifications are allowed for the app.

### Dictionary data (offline)

The bundled dictionary is built from **[OpenRussian.org](https://en.openrussian.org/)** with **[Tatoeba](https://tatoeba.org/)** examples filling gaps, at build time into `RussianWordADayApp/Resources/dictionary.sqlite`. See `scripts/README.md` and `DATA_LICENSES.md`.

### App Store privacy & legal
- In-app: **Settings → Legal & privacy** (short summary + data source attributions).
- **Public privacy policy (App Store URL):** after enabling [GitHub Pages](https://docs.github.com/en/pages/getting-started-with-github-pages/about-github-pages) for this repo (**Settings → Pages → Deploy from branch `main` → `/docs`**), use:
  - **https://thetoastykitten.github.io/word-a-day/privacy-policy.html**
- Source files: `docs/privacy-policy.html` (hosted page) and `docs/PRIVACY_POLICY.md` (same text for the repo).
- **Privacy manifest**: `RussianWordADayApp/PrivacyInfo.xcprivacy` (`NSPrivacyTracking` false; UserDefaults / `CA92.1` only).
- **Export compliance**: `ITSAppUsesNonExemptEncryption` is `false` in `project.yml`.
- **App Privacy questionnaire** (App Store Connect): answer **“No, we do not collect data from this app”** if that matches your build (no analytics/ads; dictionary offline; local notifications only; no data sent to developer servers). Align answers with sections 1–5 of the hosted policy.
